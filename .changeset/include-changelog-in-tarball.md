Include `CHANGELOG.md` in the published npm tarball.

Until now, both packages defined a `files` field in `package.json` that
listed only `dist`, `LICENSE`, `NOTICE.md` (and `README.md` for `share`).
Because `files` is an allow-list, the `CHANGELOG.md` generated by
changesets next to each `package.json` never made it into the tarball.
Consumers who relied on `npm view` or on the installed tarball had no
visibility into the changelog and had to navigate to GitHub to find it.

Adding `CHANGELOG.md` to the `files` array on both packages makes the
changelog ship with the tarball, so downstream consumers can see what
changed (and, in particular, whether a release is marked as BREAKING)
without leaving npm or their `node_modules`.

Also adds an initial `packages/share/CHANGELOG.md` covering 0.1.0 and
0.2.0, copied from the historical entries in the monorepo root changelog,
so the share package no longer ships an empty changelog on its first
post-fix release.

No runtime change. Pure packaging fix.
